# Quality Loop

Test stories → find issues → diagnose root causes → fix with review gates → verify.

## Step 1: Parse Intent

Extract from user message:
- **Area/topic**: app focus (feed, admin, campaign, sharing)
- **Iteration count**: rounds requested (default 5, max 3 per session)
- **Resume intent**: "continue", "resume", "pick up", "where we left off"

Count > 3 → acknowledge, explain: "Will run 3 this session (context budget). Resume for rest."

## Step 2: Check for Resume

If resume intent:

1. Read `tmp/quality-loop/state.json` — get branch, completed iterations, next story
2. **Lock check**: `state.lock` exists + `stoppedReason` set → stale lock, remove. No stoppedReason → warn another run may be active.
3. Verify branch exists: `git branch --list {branch}`
4. Clean working tree → `git checkout {branch}`. Dirty → ask user to commit/stash.
5. Skip to Step 6 with next unprocessed story

No state file → "No previous run found. Starting fresh."

## Step 3: Pre-flight Checks

1. **Dev servers**: curl `http://ecomitram.localhost:1355` and `http://api.ecomitram.localhost:1355/health` — expect 200. Fail → tell user, STOP.

2. **Lock check**: `state.lock` exists → warn, STOP.

3. **Working tree**: `git status --porcelain` not empty → ask commit/stash, STOP.

4. **MCP check**: Subagent navigates to `http://ecomitram.localhost:1355` via Playwright MCP, returns page title. Fails within 30s → STOP.

5. **GitHub labels**: Create if missing:
   ```bash
   gh label create quality-loop --description "Auto-generated by quality loop" --color "0E8A16" 2>/dev/null || true
   gh label create needs-review --description "Needs human review" --color "D93F0B" 2>/dev/null || true
   gh label create summary --description "Quality loop run summary" --color "0075CA" 2>/dev/null || true
   ```

## Step 4: Discover Stories

1. Glob `docs/user-stories/*.md`
2. Read first 20 lines per file — title + story IDs
3. Fuzzy match user's area against: filename (score 3) > title (score 2) > content in first 50 lines (score 1)
4. Collect stories with score >= 1, sorted by score then filename

Present matches. No matches → list all filenames, ask to refine. >15 matches → ask to narrow.

## Step 5: Initialize Run

1. `date +%Y-%m-%d-%H%M` → timestamp
2. `git checkout -b quality-loop/{area}-{timestamp}`
3. `mkdir -p tmp/quality-loop` — verify `tmp/` in `.gitignore`, add if not
4. Write lock: `echo $$ > tmp/quality-loop/state.lock`
5. Write `state.json`:
```json
{
  "runId": "{area}-{timestamp}",
  "area": "{area}",
  "branch": "quality-loop/{area}-{timestamp}",
  "totalIterations": "{user_requested_count}",
  "completedIterations": 0,
  "sessionBudget": 3,
  "stories": [],
  "stoppedReason": null,
  "githubIssues": [],
  "summaryIssue": null
}
```

## Step 6: Begin Iteration

For each iteration (up to session budget of 3):

### Context budget check

Iteration 4+ → Step 7 with reason "context-budget".

**Compaction heuristic:** Can you recall branch name + run ID without reading state.json? No → stop, save state.

### Pick next story

Next unprocessed from matched list. On second pass, skip "clean" or "skipped-all" stories.

### Phase 1: DISCOVER

Invoke `/test-stories` skill with current story filename. Extract failed criteria (lines with `✗`).

**Gate G1:** Spawn subagent with AGENT-GATE.md:
```
Gate: G1 (Discover → Triage)
Phase output: {test-stories report}
Issue context: Story {story-id}
Retry count: {N}
```
SKIP → record "untested", next iteration. RETRY → re-run /test-stories once. PROCEED → Phase 2.

### Phase 2: TRIAGE

Dispatch expert subagents based on discovery report:
- **Always:** ux-reviewer, code-standards
- **If UI/visual:** visual-reviewer
- **If mobile/responsive:** mobile-reviewer
- **If content/label/error messages:** content-reviewer
- **If slow/performance:** performance
- **If security/auth:** security
- **If error/crash:** error-handling

Each expert subagent gets: agent instructions + discovery report + story content.

After experts return → spawn triage validator (AGENT-TRIAGE.md) with story ref + discovery report + all expert outputs.

**Gate G2:** SKIP (no issues survived) → record "clean", next iteration. RETRY → re-run triage. PROCEED → Phase 3.

### Phase 3: DIAGNOSE

**Issue cap:** Max 3 issues per iteration (highest severity first). Remaining → GitHub issues.

**Retry tracking:** `issueRetryCount` map. Count >= 3 for any issue → skip to GitHub issue.

For each issue (up to 3): spawn TWO parallel investigators (AGENT-DIAGNOSE.md):
- **Investigator A**: perspective FRONTEND
- **Investigator B**: perspective BACKEND

After both return → spawn diagnosis validator (AGENT-DIAGNOSE-VALIDATOR.md) with issue + both investigation outputs.

**Gate G3:** SKIP → GitHub issue, next issue. RETRY → re-run investigators with more context. PROCEED → Phase 4.

### Phase 4: FIX

Spawn AGENT-FIX.md with confirmed diagnosis + story ref + branch name + project root. Record commit hash.

Then spawn TWO parallel reviewers:

**Code Reviewer** (code-standards agent): review `git diff HEAD~1`, check fix addresses root cause (not symptom), no scope creep. Return APPROVE/REJECT.

**Product Expert** (ux-reviewer agent): review `git diff HEAD~1`, evaluate UX improvement, check for regressions, completeness. Return APPROVE/REJECT.

**Gate G4:** Both APPROVE → Phase 5. Either REJECT + retry budget → re-run fix with feedback. Retry exhausted → `git revert {commit-hash} --no-edit` for each tracked commit, GitHub issue, next issue.

### Phase 5: VERIFY

Re-run `/test-stories` on same story.

**Gate G5:** Pass → Phase 6. Flake-looking failure → RETRY once. Consistent failure → revert all iteration commits (`git revert {hash} --no-edit`), GitHub issue.

### Phase 6: REPORT

Update state.json with iteration results:
```json
{
  "id": "{story-id}",
  "status": "fixed | clean | untested | skipped-all | partial",
  "issuesFound": 0,
  "issuesFixed": 0,
  "issuesSkipped": 0,
  "issuesDeferred": 0,
  "commits": ["abc1234"],
  "issueAttempts": { "issue-title": 2 }
}
```

Log to terminal:
```
Iteration {N}/{total}: {story-id}
  Issues found: X | Fixed: Y | Skipped: Z
  Commits: {list}
```

## Step 7: End of Run

1. **Update state:** Set `stoppedReason` ("completed" | "context-budget" | "no-more-stories"), `completedIterations`

2. **Regression sweep** (only if <3 iterations ran): run `/test-stories` on 1-2 adjacent untested stories in same area. Failures → regression warnings in summary. Don't fix, just report.

3. **Push branch:** `git push -u origin {branch}`

4. **GitHub issues for skipped items:**
   ```bash
   gh issue create \
     --title "[quality-loop] {story-id}: {issue summary}" \
     --label "quality-loop,needs-review,{area}" \
     --body "..."
   ```

5. **Summary issue:**
   ```bash
   gh issue create \
     --title "[quality-loop] Run summary: {area} ({date})" \
     --label "quality-loop,summary" \
     --body "..."
   ```
   Body: branch link, iterations completed/total, per-story results table, commit links, skipped-item issue links, stats.

6. **Remove lock:** `rm -f tmp/quality-loop/state.lock`

7. **Print summary:**
   ```
   Quality Loop Complete: {area}
   Branch: quality-loop/{area}-{timestamp}
   Iterations: {completed}/{total}
   Issues: {found} found, {fixed} fixed, {skipped} skipped
   Summary: {github issue URL}

   {if stopped early:}
   Resume with: "continue quality loop"
   ```

## Rules

- Never open browser yourself — /test-stories and subagents handle browser interaction
- Never skip phases or gates
- Stop cleanly on context compaction — never proceed after
- Phase 3 exists for root causes — never fix symptoms
- "No issues" is valid outcome — don't invent problems
- Main context stores only summary text from subagents, not full outputs
- Sequential stories only — one iteration at a time
- Every fix cross-checked by agents that didn't write it
